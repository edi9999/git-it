The Challenge:

Learn about bisecting, a useful technique to find when a bug first occured

Step: A user of our software just reported a bug

One of our user has reported us a bug : When you enter the following numbers into a file: (file.txt)

25
25
50

and run

$ node index.js file.txt

The result is not 100 , which is 25+50+50

Step: Find the source of the issue

Whenever you use git history, they is a great feature that makes it easy to find what commit introduced a bug The feature is called `git bisect` With git bisect, you can Use binary search to find the commit that introduced a bug

You first have to start git bisect, it is as easy as

$ git bisect start

You then have to say which commits are good and which are bad. Bad means that the bug is present. Good means that the bug is not present

$ git bisect good `commit`

$ git bisect bad `commit`

After these two initial commands, git will start to do a binary search and will ask you whether the commit is good or bad. Just run the test that we created in the first step, and continue to run `git bisect good` or `git bisect bad` depending on if the bug is present or not on that commit

Whenever git-bisect is finished, it will output : `commit` is the first bad commit

Verify with

git-it verify
