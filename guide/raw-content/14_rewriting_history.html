The Challenge:

Learn about interactive rebase, a useful technique to edit your git history

Configuring git-editor

When we do interactive rebase, git will open the editor of our choice to edit the history

We have to do some configuration to edit the editor that is used

See this stackoverflow question to see how to configure your editor under windows

See this github article to see how to configure your editor under other platforms

Understanding interactive rebase

Interactive rebase will present you with a list of commits.

If you just close your editor without editing anything, git will not change anything to your repository

You have multiple choices : you have to change the word `pick` to do something else with a commit

You can do one of the following:

  • reword : to rename a commit message
  • squash : will put the commit with the commit over it

Starting interactive rebase

To start interactive rebase, write:

$ git rebase --interactive `start-commit`

The objective is to have only the following commits :

  • Add documentation
  • Add code
  • Initial commit

You can then choose what to do with each commit

To edit even the first commit, use `git rebase --interactive --root`
